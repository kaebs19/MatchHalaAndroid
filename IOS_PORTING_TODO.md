# قائمة نقل تحسينات Android إلى iOS

> آخر تحديث: 2026-06-28
> المرجع: كل ما نُفِّذ في كود Android (Kotlin/Compose) خلال جلسات تطوير نظام التقييد/المراجعة + تحسينات الدردشة والإشعارات والأنيميشن.

---

## ✅ الجزء الجاهز على السيرفر (يعمل للنسختين تلقائياً)

هذه التغييرات في `MatchHalaApi` **منشورة وحيّة**، ويكفي أن يستهلكها تطبيق iOS — لا حاجة لعمل خادم جديد:

| # | الـ Endpoint / الحدث | الوصف |
|---|----------------------|-------|
| 1 | `POST .../messages` يُرجع **201** مع `messagingLocked` | عند التقييد لا يُرجع 403 — بل كائن لعرض حوار |
| 2 | `GET .../messages/:id` يُرجع `data.messagingRestriction` | بيانات بانر التقييد (نوع/سبب/مدة/`lockedUntil`) |
| 3 | `GET /api/mobile/account-standing` | الحالة + سُلّم التصعيد + `reviewable` + `nonReviewableNote` + `decay` |
| 4 | `GET /api/mobile/violations-history` | المخالفات مع `maskedEvidence` + `action` + `escalationLevel` |
| 5 | `POST /api/appeals` (actionType=`restriction`) | المراجعة تستخدم نظام الاستئناف الموحّد |
| 6 | إشعار push استباقي عند مخالفة 4/5 | `type: promo_warning` |
| 7 | `messagingRestrictionWatcher` | فك التقييد المنتهي تلقائياً (كل دقيقة، instance 0) |
| 8 | حدث socket `messaging-restriction-lifted` | بثّ فوري عند رفع التقييد |
| 9 | حدث socket `appeal-message` | ردّ المشرف اللحظي على الاستئناف |

**ما يلزم iOS:** فقط قراءة هذه الحقول/الأحداث وعرضها — المنطق الخلفي جاهز.

## 📱 الجزء الذي يحتاج برمجة واجهة في iOS (Swift/SwiftUI)

### A. نظام تقييد المراسلة (الترويج الخارجي)
- [ ] **شريط مقفل** داخل المحادثة (منع الكتابة/الصور/التسجيل) عند `messagingRestriction`
- [ ] **بانر التقييد** أعلى المحادثة (نوع + سبب + مدة)
- [ ] **بانر عام** في شاشة الاكتشاف (قابل للطي + تفاصيل + زر «مراجعة»)
- [ ] **عدّاد حيّ** يتناقص من `lockedUntil`/`until`
- [ ] **حوار الحجب** (`messagingLocked` / `externalPromoBlocked`) مع شريط المخالفات + الأنماط المكتشفة (`detectedPatterns`)
- [ ] **تحذير استباقي** عند الكتابة (كشف كلمات محلي)
- [ ] منطق **«غير قابلة للمراجعة»** (`reviewable=false` للصور الإباحية/الأسماء) → إخفاء زر المراجعة + رسالة
- [ ] التسمية: **«مراجعة»** في iOS أم الإبقاء على «استئناف»؟ (في Android = مراجعة)

### B. حالة الحساب وسجل المخالفات
- [ ] **شاشة «حالة حسابي»**: شارة (🟢🟡🟠🔴) + شريط المخالفات + سُلّم التصعيد المرئي + بطاقة الاسترداد
- [ ] **سجل المخالفات المطوّر**: فلترة بالنوع + الكلمة المُقنَّعة + شارة الإجراء
- [ ] **شارة الحالة** بجانب «الحساب» في الإعدادات

### C. طلباتي (المراجعات/الاستئنافات)
- [ ] **شاشة «طلباتي»**: نموذج تقديم + الطلبات السابقة + روابط سريعة (الشروط/الخصوصية/اتصل بنا/إنستا)
- [ ] **شاشة تفاصيل الطلب**: محادثة مع المشرف + ردّ + استماع لـ socket `appeal-message` (ردّ لحظي)
- [ ] الاستماع لـ socket `messaging-restriction-lifted` → فك القفل فوراً + تنبيه

### D. شاشة الإيقاف/الحظر (Suspension)
- [ ] **نوع الإجراء الديناميكي** (تقييد/إيقاف مؤقت/حظر دائم حسب `isPermanent`+`level`)
- [ ] **قائمة الاستئنافات السابقة** داخل الشاشة (عبر `/api/appeals/my`)
- [ ] روابط سريعة + إنستا
- [ ] **تأكيد قبل الخروج** + تسمية «تسجيل خروج» + زر ثانوي غير بارز

### E. جودة الدردشة
- [ ] **التمرير لآخر رسالة** عند فتح المحادثة (أو أول غير مقروءة)
- [ ] **تمرير تلقائي للرسالة الجديدة** (رسالتي دائماً / الواردة لو بالأسفل) + زر «رسائل جديدة»
- [ ] **السحب للردّ** (Swipe-to-reply)
- [ ] **تجميع الرسائل** المتتالية لنفس المرسل
- [ ] **زر النزول للأسفل** الدائم
- [ ] **اهتزاز** (Haptic) عند الضغط المطوّل/السحب

### F. الإشعارات
- [ ] **صور دائرية** (ملء الدائرة / `ContentScale.Crop` في iOS = `.scaledToFill().clipShape(Circle())`)
- [ ] crossfade + placeholder + بديل عند الفشل
- [ ] سلاسة التمرير + تحجيم الصور

### G. الأنيميشن والصقل
- [ ] **انتقالات الشاشات** (slide/fade)
- [ ] **حركة الشريط السفلي** (تكبير الأيقونة المختارة)
- [ ] **مؤشر «يكتب» المتموّج** (نقاط متحركة)
- [ ] **تفاعل ينطّ** (pop-in للإيموجي)
- [ ] **Shimmer** متموّج للهياكل
- [ ] **عدّاد أرقام متحرك**

### H. المحتوى الحساس (من جلسات سابقة — تحقّق من حالته في iOS)
- [ ] فقاعة محتوى حساس مموّهة + كشف عند الضغط (`/reveal`)
- [ ] مدّة كشف 30 ثانية + محادثات موثوقة (auto-reveal)
- [ ] شاشة «تفضيلات المحتوى»

## ملاحظات
- **الأولوية المقترحة:** A → C → B (جوهر نظام التقييد) ثم E (تجربة الدردشة) ثم G/F (الصقل).
- iOS الحالي (نسخة 7.x) لديه بالفعل: نظام الاستئناف، حوار الترويج الخارجي، شاشة المحتوى الحساس — راجع ما هو موجود قبل إعادة البناء.
- كل الحقول الخلفية موثّقة في كود Android المقابل (نفس أسماء الحقول في الـ JSON).
